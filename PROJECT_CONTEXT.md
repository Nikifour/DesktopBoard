# Desktop Board - контекст проекта

Файл для быстрого продолжения в следующих сессиях. Перед работой по проекту сначала прочитать его, затем при необходимости `README.md` и ключевые файлы.

## Где проект

- Основная папка: `E:\Coding Projects\desktop-board`
- Стек: Electron + vanilla HTML/CSS/JS
- Electron entry: `main.js`
- Preload bridge: `preload.js`
- UI: `src/index.html`
- Стили: `src/styles.css`
- Логика доски: `src/renderer.js`
- Состояние пользователя хранится в Electron `app.getPath("userData")`, файл `board-state.json`

```powershell

```

## Сборка EXE

- Сборщик: `electron-builder`.
- Команда: `npm.cmd run dist:win`.
- Выходная папка: `dist`.
- Артефакты текущей сборки:
  - `dist\Desktop Board 0.1.0.exe` - portable exe без установки;
  - `dist\Desktop Board Setup 0.1.0.exe` - NSIS installer;
  - `dist\win-unpacked\Desktop Board.exe` - распакованная сборка для быстрой проверки.
- В `package.json` задан `electronDist: node_modules/electron/dist` и `electronVersion: 41.2.1`, чтобы builder использовал уже установленный Electron и не скачивал Electron zip с GitHub.
- В npm-скрипте `dist:win` задано зеркало `ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/`, потому что прямые загрузки GitHub release assets на этой машине падали по timeout.
- Для MVP в `win.signAndEditExecutable` стоит `false`, чтобы не требовать загрузку `winCodeSign` и не зависеть от code-signing. Сборка не подписана, поэтому SmartScreen может предупреждать на других устройствах.
- Иконка приложения пока не задана, используется дефолтная Electron icon.

## Что это за приложение

Desktop Board - приложение-доска для Windows, работающее как полноэкранный desktop overlay. Цель: личная рабочая поверхность поверх/вместо рабочего стола с заметками, задачами, расписанием, медиа, URL-элементами, группами, zoom/pan и локальным сохранением.

Пока это не настоящий wallpaper-layer под иконками Windows. Текущий подход - практичный Electron overlay/MVP.

## Реализовано

- Полноэкранное Electron-окно без рамки.
- Локальное сохранение состояния доски.
- Hotkeys:
  - `Ctrl+Alt+B` показать/скрыть окно.
  - `Ctrl+Alt+L` закрепить/редактировать доску.
- Zoom колесом мыши вокруг курсора.
- Pan камеры средней кнопкой мыши.
- Правая кнопка открывает контекстное меню.
- Контекстное меню:
  - пустое место: создать заметку, задачи, расписание, картинку, видео, URL/сайт, группу, открыть настройки;
  - элемент: переименовать, дублировать, изменить URL для web-элемента, включить/выключить взаимодействие с сайтом, сменить цвет хедера, сменить цвет фона, сбросить цвета, удалить;
  - выбранный набор: сгруппировать, снять выделение, удалить выбранные.
- Соединения:
  - хранятся в `state.connections`;
  - каждая точка привязки может быть `card` или свободной `point`;
  - `points` внутри connection хранит неограниченное количество путевых точек;
  - SVG-слой `.connections-layer` находится внутри `workspace`, поэтому линии масштабируются и двигаются вместе с доской;
  - режим инструмента `connectionMode`: клик по старту, затем клик по финишу;
  - финиш/старт может быть карточкой или пустым местом;
  - `selectedConnectionId` хранит выделенную линию;
  - двойной щелчок по выделенной линии добавляет waypoint в ближайший сегмент, waypoint можно перетаскивать;
  - Ctrl/Meta/Shift + left click по карточке переключает ее в мультивыборе;
  - контекстное меню выбранных 2+ карточек содержит `Соединить`; для 3+ создает цепочку;
  - в контекстном меню линии можно поменять `arrowMode` (`none`, `end`, `start`, `both`), цвет или удалить линию.
  - общий цвет новых/наследуемых соединений задается в настройках; ручной цвет линии помечает ее как индивидуальную (`customColor`) и не перетирается глобальным правилом.
  - линия рисуется двумя path: белый/черный контур по яркости фона доски и цветной тонкий path сверху.
- Левая кнопка по пустому месту + drag выделяет элементы рамкой.
- Поле групповой подложки тоже считается фоном для старта рамочного выделения; саму группу двигать за хедер/drag-grip.
- Если выделено несколько элементов, drag любого выбранного элемента двигает весь набор.
- `Delete` удаляет выбранные элементы.
- Перемещение элементов только левой кнопкой мыши за хедер/drag-зону.
- Snap-to-grid при перемещении, отключается в настройках.
- Размер сетки: `gridSize = 24`. Элементы, свободные точки соединений и путевые точки используют общий snap.
- Компактная верхняя панель на иконках с tooltip.
- Группы кнопок панели:
  - заметка / задачи / расписание;
  - картинка / видео / URL;
  - группа / настройки / закрепить / скрыть.
- Элементы:
  - заметка;
  - список задач;
  - расписание;
  - картинка;
  - видео;
  - URL/сайт через Electron `webview`, с `iframe` fallback в browser preview;
  - группа-подложка.
- В хедере каждого элемента справа отображается иконка типа элемента. Цвет иконки берется из контрастного цвета хедера (`--card-header-text`).
- Группы:
  - имеют нормальный хедер;
  - можно редактировать название;
  - можно двигать за хедер/drag-grip;
  - можно удалить через контекстное меню;
  - элементы, центр которых внутри группы, двигаются вместе с группой при drag одной группы.
- Цвета:
  - у каждого элемента есть цвет хедера и цвет фона;
  - глобальные цвета новых элементов задаются в настройках;
  - ручное изменение цвета элемента через контекстное меню делает цвет индивидуальным и не перетирается глобальными правилами.
- Настройки:
  - тема интерфейса: системная / светлая / темная;
  - язык интерфейса: системный / русский / английский;
  - цвет фона доски;
  - общий цвет соединений;
  - цвета новых элементов: хедер и фон по типам;
  - snap-to-grid;
  - hotkeys.
- Секция `Цвета новых элементов` в настройках сделана раскрывающейся и по умолчанию закрыта.
- Верхняя панель и окно настроек используют системный accent color и темную/светлую тему. Добавлен ручной переключатель темы, потому что Electron/Windows иногда некорректно отдают системную тему.

## Важные UX-решения

- Доска хранит координаты в мировых координатах, экран работает как камера: `viewport.x`, `viewport.y`, `viewport.zoom`.
- Новые элементы создаются около центра видимой области или в точке контекстного меню.
- При добавлении нового элемента приложение переходит в режим редактирования.
- В режиме закрепления блокируется редактирование текста, скрывается resize, а прозрачный слой web-элемента отключается, чтобы можно было взаимодействовать с сайтом.
- В режиме редактирования у web-элемента поверх сайта есть прозрачный слой: он позволяет выбирать элемент и открывать контекстное меню.
- Для web-элемента можно включить индивидуальный режим взаимодействия с сайтом через контекстное меню или двойным щелчком по области сайта. В этом режиме клики уходят в сайт, а хедер остается доступен для перемещения и контекстного меню. `Esc` отключает взаимодействие для всех web-элементов.
- В режиме закрепления прозрачный слой web-элемента тоже отключается, чтобы сайт был интерактивен.
- URL/web-элемент должен растягиваться вместе с карточкой: `.card[data-kind="web"] .card-body` является позиционным контейнером, `.web-frame`, `iframe` и `webview` занимают всю область карточки.
- Для Electron `webview` есть дополнительная JS-синхронизация размеров: `ResizeObserver`, `syncWebCardElement()` при resize карточки и `syncAllWebCardElements()` после изменения камеры. Это нужно, потому что guest-view иногда остается в старом размере внутри transform-слоя доски.
- URL вводится через собственное модальное окно `urlModal`, не через `window.prompt`, потому что native prompt в Electron может не открываться/падать без видимой ошибки.
- Правая кнопка мыши не используется для pan.
- Хедер карточки используется только для названия и перетаскивания. Удаление и цвета вынесены в контекстное меню.
- У группы пока нет жесткого членства элементов. Групповое перемещение работает геометрически: если центр элемента внутри подложки в момент начала drag группы, элемент двигается вместе с группой.

## Известные ограничения

- Нет tray icon.
- Нет автозапуска Windows.
- Installer/portable build есть, но пока без цифровой подписи.
- Нет настоящего встраивания в desktop wallpaper/WorkerW.
- Нет явного group membership / ungroup / nested groups.
- Нет undo/redo.
- Нет multi-board/workspaces.
- Нет поиска по заметкам.
- Медиа хранятся как file URL/path; если файл переместить, карточка может потерять источник.
- Не все сайты разрешают встраивание. В Electron `webview` работает шире, чем `iframe`, но отдельные сайты все равно могут блокировать авторизацию, popups или embedded-режим.

- Перед финалом запускать:

```powershell

```

- После изменения UI перезапускать только Electron-процессы этого проекта:

```powershell
Get-Process electron -ErrorAction SilentlyContinue | Where-Object { $_.Path -like 'E:\Coding Projects\desktop-board\node_modules\electron\dist\electron.exe' } | Stop-Process -Force
Start-Process -FilePath npm.cmd -ArgumentList @('run','dev') -WorkingDirectory 'E:\Coding Projects\desktop-board' -WindowStyle Normal
```
